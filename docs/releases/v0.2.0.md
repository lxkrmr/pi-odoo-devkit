# osmo v0.2.0

Date: 2026-03-11

## Highlights

- Automation contract clarified and documented as v1 (`docs/cli-contract.md`).
- Contract boundary is explicit (`automation` vs `human-ops`) and discoverable via `osmo help --output json`.
- `wizard` and `reset-project-path` now support full automation contract flags:
  - `--output json`
  - `--describe`
  - `--dry-run`
- `components --output json` now exposes structured requirement diagnostics:
  - `reason_code`
  - `requirement_failures[]`
- `doctor --output json` now exposes structured machine guidance:
  - `checks_structured[]`
  - `recommendations_structured[]`
  - while keeping legacy `checks[]` and `recommendations[]`

## Quality gates

- Added targeted CLI contract tests (`scripts/test-cli-contracts.sh`).
- Added golden JSON snapshot tests (`scripts/test-cli-golden.sh`).
- Smoke now runs both targeted and golden contract checks.
